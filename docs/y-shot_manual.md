# y-shot 操作マニュアル

**Web Screenshot Automation Tool**
対象: 社内作業者向け / バージョン: 2.5

---

> **テストケースの保存について**
> ソフトウェアの改修が見込まれますが、テストケースさえ保存しておけば撮り直しが可能です。
> プロジェクトエクスポート（AppBarの ↑ アイコン）で `.yshot.json` として書き出しておいてください。

## 1. 初期設定

### 1-1. 動作環境

- **OS**: Windows 10 / 11
- **Python**: 3.10 以上（ソースから実行する場合）
- **Chrome**: 最新版推奨

### 1-2. 設定ダイアログ

AppBar右上の **歯車アイコン** をクリックして設定ダイアログを開きます。

| 項目 | 説明 |
|------|------|
| **Basic認証ID / パスワード** | 開発環境などBasic認証が必要なサイトで設��� |
| **出力フォルダ** | スクリーンショ��トの保存先 |
| **ヘッドレスモード** | ONにするとブラウザを表示せずに実行（高速） |
| **HTMLソース保存** | ONにするとスクショと一緒にHTMLソースを保存（y-diffでの比較用） |
| **ステップ削除時に確認する** | OFFにするとステップ削除の確認ダイアログをスキップ（一括削除時に便利） |

## 2. プロジェクト管理

y-shotはプロジェクト単位でテストデータを管理します。初回起動時に「default」プロジェクトが自動作成されます。

### 2-1. プロジェクトの切替・作成

画面上部の **プロジェクトドロップダウン** で切替できます。

- **＋ボタン**: 新規プロジェクト作成
- **⋮メニュー**: プロジェクトの編集・削除

### 2-2. プロジェクト設定

プロジェクト編集（⋮メニュー → 編集）で以下を設定できます:

- **プロジェクト名**: わかりやすい名前
- **プロジェクトURL**: 設定すると全テストケースでこのURLが起点になる（ページ・テスト個別のURLより優先）

### 2-3. データの保存場所

各プロジェクトのデータは `projects/プロジェクト名/` フォルダに自動保存されます（2秒間隔の自動保存）。

```
projects/
├── projects.json          ← プロジェクト一覧
├── default/
│   ├── y_shot_config.ini
│   ├── y_shot_tests.json
│   ├── y_shot_pages.json
│   ├── y_shot_patterns.json
│   └── y_shot_selectors.json
├── お問い合わせテスト/
│   └── ...
└── ...
```

### 2-4. プロジェクトのエクスポート / インポート

- **エクスポート**: AppBarの **↑ アイコン** → `.yshot.json` として保存
- **インポート**: AppBarの **↓ アイコン** → ファイルを選択

インポート時の取込方法:

| モード | 動作 |
|--------|------|
| **置換** | 現在のプロジェクトを上書き |
| **マージ** | 現在のプロジェクトにデータを追加（IDは自動で重複回避） |
| **新規プロジェクト** | インポートデータで新しいプロジェクトを作成 |

画面左上のページドロップダウンの横にある **＋ボタン** でページを追加します。
**⋮メニュー** からページの編集・複製・並び替え・削除ができます。

| 項目 | 説明 |
|------|------|
| **ページ名** | テスト対象の画面名（例: 「お問い合わせ入力画面」） |

## 5. 要素ブラウザでステップを記述する

画面右側の **要素ブラウザ** でテスト対象ページの要素を確認し、ステップを組み立てます。

### 5-1. ページを読み込む

1. URL欄にテスト対象のURLを入力（ページ切替で自動入力されます）
2. **読込** ボタンをクリック
3. Chromeが起動し、ページの全要素が一覧表示されます

### 5-2. 要素を選択してステップ追加

一覧から要素をクリックすると、ブラウザ上でハイライトされます。
下部のアイコンでステップを追加します:

| アイコン | 機能 | 用途 |
|---------|------|------|
| 📝 | **入力** | テキストフィールドに値を入力 |
| 🖱 | **クリック** | ボタン・リンク・チェックボックスをクリック |
| ↗ | **ホバー** | マウスホバー（メニュー展開等） |
| ⏬ | **選択** | セレクトボックスの値��選択 |
| 📋 | **全パターン** | select/radioの全選択肢を自動パターン化 |
| 💾 | **値取込** | 現在のフォーム入力値を一括取込 |
| 🔍 | **セレクタテスト** | CSS/XPathセレクタの動作確認 |
| ℹ️ | **要素詳細** | 選択要素の全属性を表示 |
| 📎 | **セレクタをコピー** | CSSセレクタをクリップボードにコピー |
| 📎 | **XPathをコピー** | XPathをクリップボードにコピー |

### 5-3. XPath対応

要素ブラウザはCSSセレクタに加え、**XPath** にも対応しています。

- CSSセレクタで一意に特定できない要素は、自動的にXPathが生成されます
- XPathセレクタは `//` で始まります（例: `//button[normalize-space()='送信']`）
- テキストや属性値を使った柔軟な要素指定が可能です

### 5-4. 要素ブラウザの便利機能

| 機能 | 説明 |
|------|------|
| **非表示要素の表示** | 「非表示」チェックで非表示要素も一覧に表示（オレンジ色で表示） |
| **検索** | タグ、type、id、name、ヒント、セレクタで絞り込み |
| **並び替え** | DOM順 / タグ順 / type順 / id/name順 |
| **DOM再取得** | ページ遷移せずに現在の状態でDOM要素を再取得 |
| **iframe対応** | iframe内の要素も自動検出・ラベル付きで表示 |
| **セレクタ履歴** | 読み込んだURLの要素情報を最大50件保存 |

### 5-5. よく使うステップの組み方

**基本的なフォーム入力テスト:**
```
1. 入力: #name ← テスト太郎
2. 入力: #email ← test@example.com
3. クリック: input[type="submit"]
4. スクショ: 表示範囲
```

**ページ遷移を含���テスト:**
```
1. 入力: #name ← テスト���郎
2. クリック: .submit-btn
3. 待機: 2秒
4. スクショ: 表示範囲        ← 確認画面
5. クリック: .confirm-btn
6. 待機: 2秒
7. スクショ: 表示範囲        ← 完了画面
```

### 5-6. ステップの手動追加・編集

中央の **＋ボタン** からステップを手動追加することもできます。
各ステップのコピーアイコンでコピー、鉛筆アイコンで編集、ゴミ箱アイコンで削除できます。
コピーしたステップは **貼り付け** ボタン（＋ボタンの隣）で追加できます。別のテストケースへの貼り付けも可能です。

**ステップの種類一覧:**

| 種類 | 説明 |
|------|------|
| 入力 | テキストフィールドに値を入力。入力モード: 上書き/追記/クリアのみ |
| クリック | 要素をクリック |
| ホバー | 要素にマウスオーバー（ホバーメニュー展開用） |
| 選択 | セレクトボックスの値を選択 |
| 待機 | 指定秒数待つ |
| 要素待機 | 指定要素が��示されるまで待つ（最大N秒） |
| スクロール | 要素へスクロール / ピクセル指定 / 先頭に戻る |
| スクショ | スクリーンショット撮影（後述） |
| 戻る | ブラウザバック + 待機 |
| 更新 | F5更新（ページリロード）+ 待機 |
| アラートOK | ブラウザの確認ダイアログ（`window.confirm`）でOKを押す |
| アラートキャンセル | ブラウザの確認ダイアログでキャンセルを押す |
| ナビゲーション | 指定URLに遷移（`{パターン}` でパターン値の置換も可能） |
| セッション削除 | Cookie・セッションをクリア（ログアウト状態にする） |
| 見出し | ステップのセクション分け（折りたたみ可能） |
| コメント | メモ（実行時はスキップ） |

### スクリーンショットモード

| モード | 説明 |
|--------|------|
| **表示範囲** | ブラウザの表示領域のみ撮影（デフォルト） |
| **ページ全体（縦長）** | ページ全体をCDP経由で縦長キャプチャ（最大16384px） |
| **要素のみ** | 指定要素だけを撮影 |
| **要素+余白** | 指定要素の周囲に余白を含めて撮影（Pillow必須） |

> **アラートOK / アラートキャンセルの使い方**
> ダイアログを発生させるクリックステップの**直後**に配置してください。
> 最大10秒間ダイアログの出現を待機します。
>
> ```
> クリック: li#complete span   ← confirm()が発生するボタン
> アラートOK                   ← ダイアログでOKを押す
> スクショ: 表示範囲           ← 送信後の画面
> ```

## 6. テストを実行する

画面下部のボタンで実行します。

| ボタン | 動作 |
|--------|------|
| **選択テスト実行** | 選択中の1テストケースのみ実行 |
| **ページ実行** | 現在のページのテストケースをすべて実行 |
| **全テスト実行** | 全ページの全テストケースを実行 |

実行中はプログレスバーとログでリアルタイムに進捗が表示されます。**中断** ボタンで途中停止可能です。

完了後、**出力フォルダを開く** ボタンで結果を確認できます。

### 出力内容

```
screenshots/
├── YYYYMMDD_プロジェクト名/
│   ├── 1_ページ名/
│   │   ├── 001_1-1_テスト名_ss1.png
│   │   ���── 001_1-1_テスト名_dom.html    ← HTMLソース保存ON時
│   │   ├── 002_1-1_テスト名_パターン名_ss1.png
│   │   └── ...
│   ├── 2_ページ名/
│   │   └── ...
│   ├── report.html     ← HTMLレポート
│   └── evidence.xlsx   ← Excelレポート（openpyxl + Pillow 必要）
```

- スクリーンショット（PNG）
- HTMLレポート（`report.html`）: 画像ギャラリー形式
- Excelレポート（`evidence.xlsx`）: ページごとのシートにスクショ埋め込み

## 7. パターンセットの活用

同じ操作を異なるデータで繰り返す場合、パターンセットを使います。

下部ナビゲーションの **パターンセット** タブで管理します。

### パターンセットの作成

1. ＋ボタンで新規作成
2. パターンを追加（ラベル + 入力値）
3. テストケース設定でパターンセットを紐付け

### 便利機能

| 機能 | 説明 |
|------|------|
| **テンプレート** | CSV形式のテンプレートからインポート |
| **文字max** | max_lengthの境界値パターンを自動生成（max-1, max, max+1） |
| **数値max** | 半角数値版の境界値パターンを自動生成 |
| **CSVエクスポート** | パターンをCSVとして書き出し |
| **コピー＆貼り付け** | 各パターンのコピーアイコンでコピーし、「貼り付け」ボタンで追加（別セットへの貼り付けも可） |
| **全パターン自動生成** | 要素ブラウザの「全パターン」ボタンで、select/radioの全選択肢をパターン化 |

### 値の指定方法（ステップ編集時）

ステップの値指定は2段階です:

1. **手入力** or **パターン** を選択
2. パターンを選んだ場合 → どのパターンセットかを選択

※ 1テストケースにつきパターンセットは1つまでです。

## 付録A: ワイン検索のテスト方法

ワイン検索ページ（セイコーマート）はホバーメニューで検索条件が表示される特殊なUIです。

### テストケースの組み方

```
1. ホバー: a.js-menu-trigger          ← メニューをホバーで開く
2. 待機: 1秒
3. クリック: input[type="checkbox"][name="type[]"][value="1"]  ← 赤ワイン
4. クリ���ク: input[type="checkbox"][name="type[]"][value="2"]  ← 白ワイン
5. クリック: input[type="submit"]      ← 検索実行
6. 待機: 2秒
7. スクショ: 表示���囲
```

### ポイント

- **ホバーステップ** を使ってメニューを開く（クリックではなくホバーで開くメニュー）
- チェックボックスは `name` と `value` の組み合わせで一意に指定
- 要素ブラウザで **非表示** チェックをONにすると、メニュー内の隠れた要素も表示されます
- 要素の **ℹ️ 詳細** ボタンで全属性を確認できます

## 付録B: ログイン必須サイトのテスト方法

ログインが必要なサイトでは、要素ブラウザのChromeウィンドウ上で手動ログインできます。

### 手順

1. 要素ブラウザでログインページのURLを読み込む
2. 表示されたChromeウィンドウで **手動ログイン** する
3. ログイン後、Chrome上で目的のページに **手動で移動** する
4. y-shotに戻って **DOM再取得** をクリック → 現在のページの要素が取得される

### テストケースでの自動ログイン

テスト実行時のログインはステップで自動化できます:

```
1. 入力: #username ← admin
2. 入���: #password ← ********
3. クリ���ク: input[type="submit"]
4. 待機: 3秒
5. ナビゲーション: https://example.com/admin/dashboard
6. スクショ: 表示範囲
```

**セッションリセットが必要な場合:**
```
1. セッション削除                ← Cookie・セッションクリア
2. ナビゲーション: https://example.com/login
3. 入力: #username ← admin
4. ...
```
