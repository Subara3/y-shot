y-shot v2.1 簡易説明書

Web Screenshot Automation Tool — 最低限これだけ知っていれば使える

y-shot とは

Webフォームの入力チェックなどで必要な大量のスクリーンショットを自動で撮るツールです。
「この項目に○○を入力 → スクショ」を何十パターンも手作業で撮る代わりに、ステップとパターンを設定するだけで全自動で実行します。

画面は大きく3列に分かれています：

領域役割
左: ページ＆テスト一覧ページ（画面単位）とテストケースを管理。ドラッグで並び替え可。
中央: ステップ＆ログ選択中テストの操作手順（ステップ）を表示・編集。下部にログ。
右: 要素ブラウザ実際のWebページを読み込み、フォーム要素を一覧表示。ここから簡単にステップを追加できる。

下部のタブで「テストケース」と「パターンセット」を切替えます。

クイックスタート（5分で最初のスクショ）

1ページのURLを設定する
左上の「ページ」ドロップダウン横の ✏（編集）ボタンをクリック。
「起点URL」にテスト対象のURLを入力して OK。
※ Basic認証が必要な場合は AppBar の ⚙設定 で認証情報を入力。
2要素ブラウザでページを読み込む
右パネルの「URL」欄に同じURLを入力（または「ページURLを使う」ボタン）→ 読込。
ブラウザが開き、ページ上のフォーム要素が一覧表示されます。
3テストケースを作る
左パネルの「テストケース」の ＋ ボタンでテストを追加。
4ステップを追加する
要素ブラウザの一覧から操作したい要素をクリック（行が選択される）→
下の [入力] [クリック] [選択] ボタンで、そのステップが中央に追加されます。
最後に中央の ＋ から「スクショ」ステップを追加してください。
5実行する
右下の [選択テスト実行] をクリック。自動でブラウザが開き、入力→スクショが実行されます。
完了後、「出力フォルダを開く」でスクショ画像を確認できます。
これだけで1件のスクショが撮れます。次に「パターン」を使うと、同じ操作を何十通りもの入力値で繰り返せます。

パターンセットで大量パターンを自動化

「パターンセット」は入力値のリストです。例えば「未入力」「半角スペースのみ」「絵文字含む」など、入力チェック用の値を一括登録しておけます。

パターンセットの作り方

1下部タブを「パターンセット」に切替え → 左の ＋ で新規作成（名前を入力）
2右側の [追加] で1件ずつ追加。または：
  • [テンプレート] → 「入力チェック_基本」を選ぶと定番パターンが一括追加
  • [文字max] / [数値max] → max_length の境界値を自動生成
  • [テンプレート] 画面の上部からCSVファイルを直接インポートも可能
3タブを「テストケース」に戻し、テスト名横の ✏ → 「パターンセット」で先ほどのセットを選択

これで実行すると、パターンセット内の全値で自動的にテストが繰り返されます。
ステップの「値」に {パターン} と入っている箇所が、各パターンの値に置き換わります。

値入力の活用：要素ブラウザの [値取込] ボタンを使うと、現在ブラウザに入力されているフォーム値を丸ごとステップとして取り込めます。手動で1回入力した状態をそのままテストケースにできるので、初回セットアップに便利です。

モード動作

モード動作

プロジェクトの共有（エクスポート/インポート）

AppBar の 📤（エクスポート） で、全ページ・テスト・パターンセットを1つの .yshot.json ファイルに保存できます。

このファイルを他の人に渡して 📥（インポート） してもらえば、同じテスト設定を共有できます。

インポート方法動作
置換現在のデータを完全に上書き
マージ現在のデータに追加（ID衝突は自動回避）

設定（⚙ボタン）

項目説明
Basic認証ID / パスワードテスト対象がBasic認証付きの場合に入力
出力フォルダスクショの保存先（日時フォルダが自動作成される）
ヘッドレスモードONにするとブラウザ画面を表示せずに実行（高速）
HTMLソース保存各スクショ時のHTMLソースも保存（diff比較用）

キーボードショートカット

キー動作

Delete選択中のテスト / パターンセットを削除

出力ファイル

実行が完了すると、出力フォルダ内に以下が生成されます：

ファイル内容
*.pngスクリーンショット画像（ページ別サブフォルダ）
report.html全スクショを一覧表示するHTMLレポート
evidence.xlsxスクショ付きExcelエビデンス（シート=ページ単位）

典型的な使い方の流れ

例：お問い合わせフォームの入力チェック

1 ページ編集 → URLを設定
2 要素ブラウザでページ読込 → フォーム要素を確認
3 パターンセットタブで「入力チェック_基本」テンプレートを読み込み
4 テスト追加 → パターンセットを紐付け
5 要素ブラウザから「名前」欄を選択 → [入力] → 値は「パターン」を選択
6 「送信」ボタンを選択 → [クリック]
7 ＋ → スクショステップを追加
8 [選択テスト実行] → 全パターンで自動実行！
テスト実行中は編集できません。実行中にテストやステップを変更しようとするとブロックされます。中断ボタンで停止してから編集してください。
