# y-shot

Webフォームの入力チェックを自動化するスクリーンショットツール。
テストパターンを登録し、入力→スクショ→送信→結果スクショをパターン分だけ自動で繰り返します。

## セットアップ

```bash
pip install -r requirements.txt
```

Chrome が必要です（ChromeDriver は Selenium 4.6+ で自動ダウンロードされます）。

## 使い方

```bash
python y_shot.py
```

### 基本的な流れ

1. **ページ作成**: テスト対象の画面ごとにページを作成し、起点URLを設定
2. **テストケース作成**: ページ内にテストケースを追加
3. **パターンセット** (タブ2): テスト入力値のセットを作成
4. **要素ブラウザ** (右パネル): ページの要素を取得してステップに追加
5. **実行**: 単体 / ページ単位 / 全テスト実行

### ページとテストケースの構造

```
ページ1: ログインフォーム (https://example.com/login)
  1-1 名前欄チェック (パターン: 入力チェック_基本)
    入力 #name ← {パターン}
    クリック #submit
    スクショ(fullpage)

  1-2 ブラウザバック確認
    入力(上書き) #name ← テスト太郎
    クリック #submit
    戻る (1.0秒)
    スクショ(fullpage)  ← 値が残っているか確認
    入力(追記) #name ← ★
    クリック #submit
    スクショ(fullpage)

ページ2: お問い合わせ (https://example.com/contact)
  2-1 必須チェック
    ...
```

### ステップタイプ

| タイプ | 説明 |
|--------|------|
| 入力 | フォームに値を入力（上書き / 追記 / クリアのみ） |
| クリック | 要素をクリック |
| 選択 | セレクトボックスの選択 |
| 待機 | 指定秒数待機 |
| スクショ | スクリーンショット撮影（表示範囲/ページ全体/要素/余白付き） |
| 戻る | ブラウザバック + 待機 |
| 更新 | F5更新（ページリロード）+ 待機 |
| アラートOK | ブラウザ確認ダイアログでOKを押す（直前のクリックの直後に配置） |
| アラートキャンセル | ブラウザ確認ダイアログでキャンセルを押す（同上） |
| ナビゲーション | 指定URLに直接移動 |
| 見出し | ステップのセクション区切り |
| コメント | メモ用 |

### URL設定

テストの起点URLはページ単位で設定します。テストケースごとに個別URLを上書きすることも可能です。

## 出力

テスト実行後、出力フォルダにページごとのサブフォルダが作成されます:

- `*.png` — スクリーンショット画像
- `report.html` — スクショ一覧のHTMLレポート
- `evidence.xlsx` — ページごとにシート分割されたExcelファイル（エビデンス用）
- `_source/` — HTMLソース保存（設定で有効化した場合）

## 機能一覧

- ページ（グループ）によるテストケース管理
- テストケースの自動ナンバリング（1-1, 1-2, 2-1 ...）
- テストケース/ステップ/パターンのドラッグ並び替え
- ページの複製・並び替え
- ステップ/パターンのコピー＆貼り付け（テスト・セット間も可）
- パターンセットのリネーム・CSVエクスポート
- max_length用の境界値パターン自動生成（文字/数値）
- 入力モード（上書き/追記/クリアのみ）
- ブラウザバック・F5更新・ナビゲーションステップ
- アラートOK/キャンセル（ブラウザ確認ダイアログ対応）
- 要素ブラウザ（検索・非表示要素表示・セレクトボックス/ラジオの全パターン自動生成）
- セレクタバンク（URL別に要素情報を保存）
- ヘッドレスモード（ブラウザ非表示で高速実行）
- Basic認証対応
- 実行モード: 単体 / ページ単位 / 全テスト
- HTMLレポート・Excelエビデンス自動生成
- 自動保存（2秒デバウンス）
- ファイルログ（log/YYYYMMDD.log）
- 中断ボタン

## exe化

```bash
build.bat
```

`dist/y-shot.exe` が生成されます。`templates/` フォルダはexeに同梱されます。

## ファイル構成

```
y-shot/
├── y_shot.py              # メインアプリ
├── requirements.txt       # 依存パッケージ
├── build.bat              # exe化スクリプト
├── test_y_shot.py         # テストスクリプト
├── templates/             # パターンテンプレート (CSV)
│   └── 入力チェック_基本.csv
├── y_shot_config.ini      # 設定ファイル (自動生成、git除外)
├── y_shot_pages.json      # ページ定義 (自動生成)
├── y_shot_tests.json      # テストケース (自動生成)
├── y_shot_patterns.json   # パターンセット (自動生成)
├── y_shot_selectors.json  # セレクタバンク (自動生成)
├── log/                   # 実行ログ (自動生成)
└── screenshots/           # スクショ出力先 (自動生成)
```
